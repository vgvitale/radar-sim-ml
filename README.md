# Airborne Radar Simulation — ML vs. Traditional Classification

Simulates an airborne radar environment, classifies aircraft contacts using a neural network and a traditional algorithm, and compares their performance.
NOTE: As making an AI-capable radar simulation is the point of this project, AI was used in this codebase. Particularly the training process, which was fully done via Claude Opus 4.7.

---

## Repository Structure

| File | Description |
|---|---|
| `radar_model.py` | Shared module — class definitions, MLP architecture, helpers |
| `01_simulate.py` | Generates the synthetic dataset |
| `02_train.py` | Trains the ML classifier |
| `03_infer.py` | ML classifier live inference loop |
| `04_attack.py` | FGSM adversarial attack on the ML classifier |
| `05_traditional.py` | Traditional (nearest-centroid) classifier live inference loop |
| `06_compare.py` | Evaluates both classifiers; saves `comparison_results.png` |
| `requirements.txt` | Python dependencies |

Generated at runtime (not committed): `dataset.npz`, `model.pt`, `scaler.npz`, `comparison_results.png`

Run from the project root in order:

```bash

python radar_sim/02_train.py       # train ML model
python radar_sim/03_infer.py       # ML live inference (Ctrl+C to stop)

python radar_sim/05_traditional.py # traditional live inference (Ctrl+C to stop)
python radar_sim/06_compare.py     # compare both classifiers + save chart
```

## Dataset

Synthetically generated by `01_simulate.py` — no external data required.

- 4,000 samples (1,000 per class), balanced
- 4 features per contact: `range_km`, `velocity_kmh`, `rcs_dbsm`, `snr_db`
- 4 classes: Commercial Aircraft, Fighter Jet, Helicopter, Drone
- Stored as `dataset.npz` (NumPy archive)

SNR is derived from the radar range equation: `SNR ∝ RCS / R⁴`

---

## Classifiers

**ML Classifier** (`02_train.py`, `03_infer.py`) — 3-layer MLP (4→32→16→4) trained with PyTorch, Adam optimizer, CrossEntropyLoss, 50 epochs.

**Traditional Classifier** (`05_traditional.py`) — nearest-centroid algorithm. For each contact, computes the normalised distance to the expected centroid of each aircraft class and assigns the nearest. No training required — centroids are derived from the known physical parameter ranges in `radar_model.py`. This is the alternative system compared in the evaluation.

---

## Replicating the Evaluation

```bash
python radar_sim/01_simulate.py
python radar_sim/02_train.py
python radar_sim/06_compare.py
```

`06_compare.py` holds out 20% of the dataset (800 samples, fixed seed 99) and runs both classifiers on the same test set. It prints accuracy, precision, recall, and F1 per class, then saves a 4-panel chart (`comparison_results.png`) showing overall accuracy, per-class F1, and confusion matrices for both classifiers.

Expected results: ML ~100% accuracy, Traditional ~89% (weakest on Helicopter due to feature overlap with other classes).
